# Структура файлов проекта

Этот файл объясняет, за что отвечает каждый основной файл в проекте.

## `Cargo.toml`
- Главный конфигурационный файл Rust-проекта.
- Содержит имя пакета, версию, edition и зависимости.
- Нужен для `cargo build`, `cargo run`, `cargo check`.

## `Cargo.lock`
- Фиксирует точные версии зависимостей.
- Нужен для воспроизводимой сборки (одинаковый результат на разных машинах).

## `src/lib.rs`
- Точка входа библиотечной части проекта.
- Публикует модули:
  - `a2_tree`
  - `binary_search`
  - `digital_sort`
- Благодаря этому другие части кода (и коллега Андрей) могут использовать инструменты через единый API.

## `src/digital_sort.rs`
- Реализация цифровой сортировки (radix/LSD) без сторонних библиотек.
- Основные функции:
  - сортировка по числовому ключу,
  - сортировка строковых ключей по байтам,
  - `radix_sort_by_selected_field(...)`.
- Поддерживает выбор поля (`id/author/name/year`) и направления (`asc/desc`).
- Отвечает только за сортировку.

## `src/binary_search.rs`
- Реализация бинарного поиска в отсортированных данных.
- Содержит:
  - generic-поиск `binary_search_by(...)`,
  - `lower_bound_by(...)`,
  - `upper_bound_by(...)`,
  - поиск диапазона совпадающих элементов `equal_range_by(...)`.
- Отвечает только за поиск по уже отсортированному массиву.

## `src/a2_tree.rs`
- Реализация дерева поиска и A2-построения по весам.
- Содержит:
  - структуру `A2Tree<T>`,
  - операции вставки и проверки наличия,
  - симметричный обход (`inorder`),
  - `build_from_sorted_by_weight(...)` для построения дерева по методике A2.
- Используется, когда нужно представить найденные/отсортированные данные в виде дерева.

## `src/main.rs`
- Демонстрационный исполняемый файл.
- Показывает пример использования сортировки, бинарного поиска и дерева.
- Это не обязательный API-файл, а пример запуска.

---

## Как это работает вместе
1. Данные приходят в виде массива ваших записей (например, `Vec<SongRecord>`).
2. Выбирается критерий сортировки и направление.
3. Применяется `digital_sort`.
4. По отсортированному массиву выполняется `binary_search`.
5. По результатам при необходимости строится `A2Tree`.

Итог: проект разделен на независимые инструменты, которые удобно вызывать из внешнего слоя (например, из backend-кода Андрея).
